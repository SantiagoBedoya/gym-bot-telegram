# Gym Bot

An AI-powered personal training assistant Telegram bot built in Go. It uses OpenAI's Responses API (agentic loop with tool use) to manage gym routines, track workout sessions, and provide progressive overload recommendations.

## Features

- **Routine management** — Create and update workout routines with exercises, sets, and rep ranges
- **Session logging** — Record completed workouts with weights, reps, and RPE per set
- **Progressive overload guidance** — AI applies a double progression model (increase reps → increase weight → deload on high RPE)
- **Exercise history** — Query past performance to inform next session targets
- **Conversation continuity** — Maintains context across messages using OpenAI's `previous_response_id`
- **Single-user** — Restricted to one authorized Telegram user

## Architecture

```
Telegram message → TelegramHandler (auth check) → OpenAIService.Chat()
    → [Agentic loop: OpenAI ↔ ToolDispatcher ↔ PostgreSQL]
    → Response sent to user, bot_state saved
```

**AI Tools available to the model:**

| Tool | Description |
|------|-------------|
| `save_routine` | Upsert a routine and its exercises |
| `get_routine` | Fetch all exercises in a routine |
| `get_exercise_history` | Recent set-by-set performance data |
| `log_session_sets` | Record a completed workout session |
| `get_last_session_summary` | Retrieve metrics from the most recent session |

## Prerequisites

- Go 1.25+
- PostgreSQL database (e.g. [Neon](https://neon.tech) serverless)
- Telegram bot token (from [@BotFather](https://t.me/BotFather))
- OpenAI API key

## Setup

**1. Clone the repo**

```bash
git clone https://github.com/SantiagoBedoya/gym-bot.git
cd gym-bot
```

**2. Install dependencies**

```bash
go mod tidy
```

**3. Configure environment variables**

```bash
cp .env.example .env
```

Edit `.env`:

```env
BOT_TOKEN=your_telegram_bot_token
OPENAI_API_KEY=sk-...
DATABASE_URL=postgres://user:password@host:5432/gymbot?sslmode=disable
AUTHORIZED_USER_ID=123456789
OPENAI_MODEL=gpt-4o
```

| Variable | Required | Description |
|----------|----------|-------------|
| `BOT_TOKEN` | Yes | Telegram bot token from BotFather |
| `OPENAI_API_KEY` | Yes | OpenAI API key |
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `AUTHORIZED_USER_ID` | Yes | Telegram user ID allowed to interact with the bot |
| `OPENAI_MODEL` | No | OpenAI model to use (default: `gpt-4o`) |

**4. Run**

```bash
go run ./cmd/bot/main.go
```

Database migrations run automatically on startup — no manual setup needed.

## Commands

```bash
# Run the bot
go run ./cmd/bot/main.go

# Build binary
go build -o gym-bot ./cmd/bot/

# Regenerate database query code (after editing SQL in internal/db/queries/)
sqlc generate

# Manage dependencies
go mod tidy
```

## Database

PostgreSQL schema managed via embedded migrations in `internal/db/migrations/`. Tables:

- `routines` — Workout routine definitions
- `exercises` — Exercises belonging to a routine
- `sessions` — Recorded workout sessions
- `session_sets` — Individual set data (reps, weight, RPE) per session
- `bot_state` — Key-value store for conversation state (`previous_response_id`)

> **Note:** Do not edit `internal/db/*.sql.go` — they are auto-generated by sqlc. Edit `internal/db/queries/*.sql` and run `sqlc generate`.
